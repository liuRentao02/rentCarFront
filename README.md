## 项目说明 — 校园租车平台（前端）

### 项目概述

独立开发了一个面向高校师生的**校园共享汽车租赁平台**前端系统，采用 **Vue 3 + Vite + Element Plus** 技术栈构建，覆盖用户端和管理端双门户，实现了从车辆浏览、在线预定、增值服务选购、支付结算到订单管理的完整业务闭环，并集成**基于 WebSocket 的实时客服聊天**功能。

### 技术栈

`Vue 3`（Composition API / `<script setup>`） · `Vite 7` · `Element Plus` · `Pinia` · `Vue Router 4` · `Axios` · `ECharts` · `STOMP.js`（WebSocket） · `SCSS` · `Quill` 富文本

### 核心职责与工作

- **双门户架构设计**：基于 Vue Router 的嵌套路由和懒加载，设计并实现了用户端（首页、车辆浏览、预定下单、支付、评价等 14 个页面）和管理端（仪表盘、车辆/订单/用户/评价/认证管理、系统配置等 17 个功能模块）两套独立的布局与导航体系，配合路由守卫实现基于角色的权限控制。
- **完整业务链路**：开发了从车辆检索 → 详情查看 → 多步预定（选取/还车站点、日期、增值服务勾选、实时价格预估）→ 在线支付（微信/支付宝/银行卡/余额）→ 订单生命周期管理（取消/支付/申请还车/续租/评价）的全流程交互。
- **实时通讯系统**：基于 STOMP over WebSocket 协议实现客服聊天功能，支持用户发起会话、管理员多会话管理、消息收发、未读标记、断线自动重连（心跳 4s，重连间隔 5s），并采用乐观更新策略提升消息发送的即时反馈体验。
- **管理仪表盘**：使用 ECharts 实现 30 天订单/收入趋势折线图、车辆出租率饼图，结合 8 项关键指标统计卡片（用户数、车辆数、订单量、营收等）和最新数据列表，为运营决策提供直观的数据透视。
- **状态管理与持久化**：使用 Pinia 管理全局应用状态（用户认证、角色权限、平台配置、车辆缓存），结合 `pinia-plugin-persistedstate` 实现关键状态持久化，保障页面刷新后登录态不丢失。
- **工程化实践**：通过 Axios 拦截器统一处理请求鉴权（Bearer Token 自动注入）、错误码映射与中文友好提示；基于 CSS 变量的暗色模式切换；平台名称、Logo、联系方式等关键信息通过后端接口动态配置；利用 Vite 的 `import.meta.glob` 实现公共组件自动全局注册。
- **交互与体验优化**：学生认证身份后享受优惠价格；多图片上传（Base64 转换）；付款结果页的 SVG 动画反馈；评价区的多维筛选、点赞与可见性控制；全站响应式布局适配。

### 项目亮点

> 独立完成从前端架构设计到全部功能开发，代码结构清晰、组件复用率高、前后端接口对接完整，具备上线级交付质量。
